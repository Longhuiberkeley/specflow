# ReqIF & Standards

## ReqIF Round-Trip (`lib/reqif.py`)

### Import Algorithm

```python
def import_reqif(archive_path: Path, root: Path) -> dict:
    # 1. Extract .reqifz (ZIP) or read .reqif (XML)
    # 2. Parse XML into ElementTree
    # 3. For each <SPEC-OBJECT>:
    #    a. Extract IDENTIFIER, LAST-CHANGE, TYPE
    #    b. Map TYPE to SpecFlow artifact type:
       - "Requirement" → requirement
       - "Architecture" → architecture (if custom type)
       - "Design" → detailed-design (if custom type)
    #    c. Extract <ATTRIBUTE-VALUE-XHTML> → body
    #    d. Extract <ATTRIBUTE-VALUE-STRING> → frontmatter fields
    #    e. Unmapped attributes → reqif_metadata dict
    # 4. Generate SpecFlow ID from ReqIF IDENTIFIER
    #    - If matches regex (e.g., REQ-001) → use as-is
    #    - Else → assign next sequential ID
    # 5. Create artifact file in appropriate directory
    # 6. Return summary: created, updated, skipped counts
```

### Export Algorithm

```python
def export_reqif(root: Path, output_path: Path) -> dict:
    # 1. Discover all artifacts (REQ, ARCH, DDD, UT, IT, QT)
    # 2. For each artifact:
    #    a. Use the artifact's SpecFlow ID directly as the ReqIF
    #       <SPEC-OBJECT IDENTIFIER="..."> (e.g., "REQ-001").
    #    b. Map frontmatter to ReqIF attributes:
       - title  → ATTRIBUTE-VALUE-STRING(ReqIF.Name)
       - body   → ATTRIBUTE-VALUE-STRING(ReqIF.Text)
       - rationale → ATTRIBUTE-VALUE-STRING(ReqIF.Description)
       - status → ATTRIBUTE-VALUE-STRING(SpecFlow.Status)
       - type   → ATTRIBUTE-VALUE-STRING(SpecFlow.Type)
       - tags   → ATTRIBUTE-VALUE-STRING(SpecFlow.Tags)
    #    c. Include reqif_metadata dict as extra string attributes
    # 3. Build ReqIF XML structure:
    #    - <REQ-IF-HEADER> with creation time, tool info
    #    - <CORE-CONTENT> with <REQ-IF-CONTENT>
    #    - <SPEC-OBJECTS> containing all artifacts
    #    - <SPECIFICATIONS> / <SPEC-HIERARCHY> tree
    # 4. Internal structure IDs (headers, type defs, attribute defs,
    #    spec hierarchies) are generated by `_new_id(prefix, seed)`:
    #       sha256(f"{prefix}:{seed}")[:10]        if seed is provided
    #       uuid4().hex[:10]                       otherwise
    # 5. Write as .reqif XML (ZIP/.reqifz packaging optional).
```

### Deterministic IDs

**Scope**: Determinism is required so that re-exporting the same SpecFlow
artifacts produces byte-equivalent ReqIF XML (important for diffs,
signatures, and ALM tools tracking identity across exchanges).

**Two ID layers**:

1. **Artifact IDENTIFIERs** — the SpecFlow artifact ID (e.g., `REQ-001`,
   `ARCH-004`) is written directly as the `<SPEC-OBJECT IDENTIFIER="...">`.
   These IDs are already stable across exports, so no UUID derivation is
   needed. On import, a ReqIF `IDENTIFIER` that matches the SpecFlow ID
   regex is reused as-is; otherwise a new sequential ID is assigned.

2. **Internal ReqIF structure IDs** — type definitions, attribute
   definitions, spec hierarchies, and header elements use
   `_new_id(prefix, seed)` (`src/specflow/lib/reqif.py:331`):
   - With a seed: `f"_{prefix}-{sha256(f'{prefix}:{seed}').hexdigest()[:10]}"`
     — deterministic across exports.
   - Without a seed: `f"_{prefix}-{uuid4().hex[:10]}"` — used only where
     determinism is not required (e.g., the top-level REQ-IF-HEADER).

**Verification**: Determinism is covered by
`tests/test_reqif_roundtrip.py::test_export_twice_same_uuids` (full-export
equivalence) and `::test_internal_id_stability` (per-call stability of
`_new_id`).

## Standards YAML Format (`lib/standards.py`)

### Clause File Structure

```yaml
id: "ISO26262-3.7"
standard: "ISO 26262"
part: "Part 3: Concept Phase"
clause: "7"
title: "Hazard Analysis and Risk Assessment"
description: "The hazard analysis and risk assessment shall..."
severity: high    # high | medium | low
priority: 1       # 1 = highest
category: safety  # safety | security | functional | process
```

### Gap Analysis Algorithm

```python
def gap_analysis(artifacts: list[Artifact], standards: list[Standard]) -> dict:
    results = {}
    for std in standards:
        covered = []
        uncovered = []
        for clause in std.clauses:
            # Find artifacts linking to this clause
            linked = [a for a in artifacts 
                     if any(l.target == clause.id and l.role == "complies_with" 
                            for l in a.links)]
            if linked:
                covered.append({"clause": clause.id, "artifacts": [a.id for a in linked]})
            else:
                uncovered.append({"clause": clause.id, "severity": clause.severity})
        
        score = len(covered) / (len(covered) + len(uncovered)) * 100
        results[std.name] = {
            "total_clauses": len(std.clauses),
            "covered": len(covered),
            "uncovered": len(uncovered),
            "score": round(score, 1),
            "gaps": uncovered,
        }
    return results
```

**Remediation Suggestions** (rule-based, per uncovered clause):

```python
REMEDIATION_MAP = {
    "safety": "Consider creating a requirement with tags: [hazard, safety]",
    "security": "Consider creating a requirement with tags: [security, threat-model]",
    "functional": "Consider creating a requirement or detailed-design",
    "process": "Consider creating a decision or story",
}

def suggest_remediation(clause: dict) -> str:
    category = clause.get("category", "functional")
    severity = clause.get("severity", "medium")
    base = REMEDIATION_MAP.get(category, REMEDIATION_MAP["functional"])
    if severity == "high":
        base += " (high severity — prioritize)"
    return base
```

## Acceptance Criteria

1. Export same artifacts twice → byte-equivalent ReqIF XML (identical
   `SPEC-OBJECT` IDENTIFIERs and identical seeded internal IDs).
2. Import valid ReqIF → correct REQ artifacts with preserved `reqif_metadata`.
3. Gap analysis score = covered / (covered + uncovered) × 100.
4. `--json` flag produces valid JSON with standard, score, covered, uncovered, gaps, and remediation fields.
5. Uncovered clauses sorted by severity (high → medium → low).
