## Extraction Protocol

The skill instructs the AI agent to follow a phased extraction protocol that minimizes hallucination risk for large documents.

### Phase 1: TOC Extraction

```
1. Read the source document (PDF/URL/text)
2. Extract the table of contents / section headings
3. Present TOC as a numbered selection menu to the user
4. User selects sections to include (or "all")
```

**Output**: Scoped list of section identifiers for extraction.

### Phase 2: Section-by-Section Chunking

```
For each selected section:
  1. Extract clause hierarchy within the section
  2. For each clause, produce: {id, title, description}
     - id: hierarchical clause number (e.g., "ISO26262-3.7.1")
     - title: clause heading text
     - description: 1-3 sentence summary (NOT full text)
  3. Preserve hierarchical numbering — do not flatten
```

**Output**: `clauses` list for the standard YAML file.

### Phase 3: Deduplication Pass

```
For each pair of adjacent sections:
  1. Compare extracted clause IDs and titles
  2. If overlapping fragments found:
     a. Merge into a single clause
     b. Prefer the more specific description
  3. Log merged pairs for user review
```

### Phase 4: Verification Layer

```
After extraction completes:
  1. Randomly select 2-3 source sections
  2. For each: compare visible headings against extracted clauses
  3. Flag if clause count mismatch
  4. Emit # VERIFY: comment in YAML:
     # VERIFY: Spot-checked sections 3.7, 4.2, 6.5 — all clauses present
```

## Pack Validation Script

`validate-pack.sh` performs deterministic structural checks:

| Check | Pass Condition |
|-------|---------------|
| `pack.yaml` exists | File present and valid YAML |
| `standards/` exists | Directory present |
| Standard YAML valid | `standard`, `title`, `version`, `clauses` keys present |
| Clause format | Each clause has `id`, `title`, `description` |
| Schema YAML valid | If `schemas/` present, each file has `type`, `prefix`, `allowed_status` |
| No empty fields | All clause strings non-empty |

Exit codes: 0 = valid, 1 = invalid.

## Pack Installation Flow

```python
def apply_pack(root, pack_name, packs_dir):
    manifest = yaml.safe_load((packs_dir / pack_name / "pack.yaml").read_text())
    
    # 1. Copy schemas (no overwrite)
    for schema_file in (packs_dir / pack_name / "schemas").glob("*.yaml"):
        dest = root / ".specflow" / "schema" / schema_file.name
        if not dest.exists():
            shutil.copy2(schema_file, dest)
    
    # 2. Create directories
    for d in manifest.get("adds_directories", []):
        (root / "_specflow" / d).mkdir(parents=True, exist_ok=True)
    
    # 3. Copy standards (no overwrite)
    for std_file in (packs_dir / pack_name / "standards").glob("*.yaml"):
        dest = root / ".specflow" / "standards" / std_file.name
        if not dest.exists():
            shutil.copy2(std_file, dest)
    
    # 4. Update config
    config["active_packs"].append(pack_name)
    config["artifact_types"].extend(manifest.get("adds_artifact_types", []))
```

## Compliance Link Validation

After pack installation, `artifact-lint` validates `complies_with` links:

1. Build set of valid clause IDs from all installed standards
2. For each `complies_with` link in any artifact, check target is a known clause
3. Flag broken links as blocking errors

## Acceptance Criteria

1. Given a 100-page PDF, the extraction protocol extracts TOC first, scopes to selected sections only
2. Spot-check verification confirms no missed clauses for sampled sections
3. Dedup pass merges overlapping fragments from adjacent sections
4. `validate-pack.sh` catches structural errors (missing fields, invalid YAML)
5. `apply_pack()` installs pack without overwriting existing project files
