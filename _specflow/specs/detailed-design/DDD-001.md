# Impact Analysis Engine

## Purpose

Implements fingerprint-based change detection, suspect flag propagation, and the 3-tier typo cascade defense. All operations are deterministic (zero LLM tokens).

## Core Algorithm: Suspect Propagation

### Function: propagate_suspects(changed_artifact_id, root)

1. Load the changed artifact's frontmatter
2. Recompute fingerprint from body content (SHA256 of Markdown after frontmatter delimiter)
3. Compare against stored fingerprint — if match, return (no change)
4. If mismatch:
   a. Update fingerprint in the changed artifact's frontmatter
   b. Bump version by 1
   c. Find all downstream artifacts: scan ALL artifacts for links where target == changed_artifact_id
   d. For each downstream artifact: set suspect: true in frontmatter
   e. For hierarchical artifacts: if child changed, set parent suspect: partial; if parent changed, flag all children
   f. Create impact-log event file

### Direction Convention
- Links point UPSTREAM (e.g., ARCH-001 has link target: REQ-001, role: derives_from)
- Propagation goes DOWNSTREAM: "who references me?" — scan for artifacts whose links.target matches the changed artifact
- This is a reverse traversal of the link graph

### Data Structures

```python
@dataclass
class ImpactEvent:
    changed: str              # Artifact ID that changed
    change_type: str          # content_modified | status_changed | created | deleted | split | merged
    fingerprint_old: str      # Previous fingerprint
    fingerprint_new: str      # New fingerprint
    update_type: str          # semantic | minor
    flagged_suspects: list    # [{artifact: str, link_role: str}]
    resolved: bool            # False initially
    resolved_by: str | None
    resolved_at: str | None
```

## 3-Tier Typo Cascade Defense

### Decision Tree

```
Fingerprint changed?
├─ No → done (no change)
└─ Yes → check update_type field in frontmatter
   ├─ update_type: minor → update fingerprint, log as minor, remove field, SKIP cascade
   ├─ update_type: semantic → update fingerprint, TRIGGER cascade
   └─ update_type not set → apply magnitude heuristic
      ├─ change ratio < 5% AND only frontmatter changed → auto-classify minor, SKIP cascade
      └─ otherwise → TRIGGER cascade (conservative default)
```

### Magnitude Heuristic (Tier 3)

```python
def classify_change_magnitude(file_path: Path, git_diff_lines: int) -> str:
    total_lines = file_path.read_text().count('\n')
    ratio = git_diff_lines / max(total_lines, 1)
    if ratio < 0.05:
        # Check if changes are only in frontmatter
        # (between first and second --- delimiters)
        return 'minor'
    return 'semantic'
```

## Split/Merge Operations

### Split: artifact A splits into A + B
1. User creates artifact B
2. System reads A's downstream links (all artifacts referencing A)
3. System presents downstream list to user for reassignment
4. User selects which links move from A to B
5. System rewrites selected link targets in downstream artifacts (A → B)
6. System logs split event in impact-log

### Merge: artifact B merges into A
1. User sets status: merged_into and merged_target: A on artifact B
2. System finds all artifacts whose links.target == B
3. System rewrites all those links: B → A
4. System logs merge event in impact-log

## Error Handling

- If artifact file is malformed (no YAML frontmatter): skip with warning, do not crash
- If link target does not exist: log warning, do not propagate to non-existent target
- If impact-log directory does not exist: create it
- If artifact is locked: queue for retry, do not skip silently
