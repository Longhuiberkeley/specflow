# Trace and Suspect Propagation Qualification

## Test Reference

`tests/test_trace_suspect.py`

## Test Cases

### TC-001: Fingerprint Change Detection
- **Input**: Modify artifact body content, run fingerprint check
- **Expected**: Changed fingerprint detected, suspect flag propagated to downstream
- **Verification**: All downstream artifacts marked suspect

### TC-002: Suspect Flag Resolution via Impact Command
- **Input**: Run `specflow impact` on artifact with suspect flag
- **Expected**: Report shows impacted artifacts, flags resolved after review
- **Verification**: Suspect flags cleared after explicit resolution

### TC-003: Typo Cascade Defense
- **Input**: Introduce typo in linked artifact
- **Expected**: 3-tier classification (cosmetic/functional/breaking) applied
- **Verification**: Correct tier assigned, propagation follows tier rules

### TC-004: Split and Merge Operations
- **Input**: Split artifact into two, then merge back
- **Expected**: Links preserved, IDs reassigned, no orphan references
- **Verification**: All link targets resolve after split and merge

### TC-005: Filesystem Locks
- **Input**: Concurrent modification attempts on same artifact
- **Expected**: Second attempt blocked or queued, no data loss
- **Verification**: Artifact integrity maintained under concurrent access

## Acceptance Criteria

1. All test cases pass deterministically
2. Tests run with no network or LLM calls
3. Tests use `tmp_path` fixtures for isolation
