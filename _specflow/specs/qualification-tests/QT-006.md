# AI Execution Boundaries Qualification

## Test Reference

`tests/test_ai_execution.py`

## Test Cases

### TC-001: Wave Grouping
- **Input**: Set of stories with dependency links
- **Expected**: Stories grouped into dependency-free waves for parallel execution
- **Verification**: No wave contains stories with inter-wave dependencies

### TC-002: Parallel Subagent Execution
- **Input**: Wave of stories with isolated context per story
- **Expected**: Each subagent receives only its story context, no cross-contamination
- **Verification**: Context isolation verified, no shared mutable state

### TC-003: Review Criteria Assembly
- **Input**: Artifact with spec links, run check command
- **Expected**: Context-specific review criteria assembled from linked specs
- **Verification**: Criteria match spec requirements, no irrelevant checks

### TC-004: Challenge Engine
- **Input**: Artifact with edge case potential, run challenge
- **Expected**: Edge cases discovered and reported
- **Verification**: Challenge output includes actionable edge cases

### TC-005: Prevention Pattern Extraction
- **Input**: Completed phase with defect history, run done
- **Expected**: Prevention patterns extracted into learned checklists
- **Verification**: Checklists contain actionable patterns from defect data

### TC-006: CI Two-Pass Validation
- **Input**: Run CI integration job on project with implemented stories
- **Expected**: Pass 1 (deterministic) runs, Pass 2 (LLM) runs if configured
- **Verification**: Both passes complete, results aggregated

### TC-007: Duplicate Detection Pipeline
- **Input**: Create artifact similar to existing one
- **Expected**: Tiered detection flags similarity at appropriate confidence level
- **Verification**: Detection pipeline runs through all tiers, results accurate

### TC-008: Dead Code Detection
- **Input**: Project with unused code paths
- **Expected**: Dead code and similar code patterns identified
- **Verification**: Report lists dead code with file locations and similarity scores

## Acceptance Criteria

1. All test cases pass deterministically
2. Tests run with no network or LLM calls
3. Tests use `tmp_path` fixtures for isolation
