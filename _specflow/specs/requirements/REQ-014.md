# Deprecation Cleanup and Schema Fixes

The system **shall** remove deprecated CLI aliases and fix schema inconsistencies.

## Deprecated CLI Aliases

The following 8 hidden aliases in `cli.py` **shall** be removed:

| Old (deprecated) | New command |
|-------------------|-------------|
| `validate` | `artifact-lint` |
| `check` | `checklist-run` |
| `impact` | `change-impact` |
| `tweak` | `fingerprint-refresh` |
| `sequence` | `renumber-drafts` |
| `verify` | `artifact-review` |
| `audit` | `project-audit` |
| `compliance` | `project-audit` |

These aliases were hidden from help but still functional. Removing them reduces maintenance burden and avoids confusion about the canonical command surface.

## Schema Fixes

1. Add `reqif_metadata` to the requirement schema `optional_fields` list (also tracked in REQ-011, but this is the minimal fix)

## Acceptance Criteria

1. None of the 8 deprecated alias names resolve as subcommands
2. Running any deprecated alias produces a clear "command not found" error
3. `reqif_metadata` appears in `.specflow/schema/requirement.yaml` optional_fields
4. `specflow artifact-lint` passes with no blocking issues after cleanup
5. All existing tests continue to pass
