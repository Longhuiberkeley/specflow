# Traceability Depth Reporting

The system **shall** provide traceability chain visualization and depth statistics as informational reports, not validation errors.

## Trace Command

A `specflow trace <ID>` command **shall** display the full traceability chain for a given artifact:

- Walk upstream: from the artifact to all standards/packs it traces to via `complies_with` and `derives_from` links
- Walk downstream: from the artifact to all implementation and verification artifacts via `refined_by`, `implements`, `verified_by` links
- Display the chain as an indented tree showing each linked artifact's ID, type, title, and status
- Support pack-added artifact types in the chain (e.g., HAZ for ISO 26262)

## Chain Report

A `specflow artifact-lint --type chain-report` check **shall** produce a survey of chain depth across all approved requirements:

- For each top-level artifact (REQ, or pack-added spec type), report the chain depth as the number of distinct link hops from standard to verification test
- Present chain depth distribution as a summary (e.g., "3 chains at depth 2, 5 chains at depth 5")
- Identify partial chains where the traceability thread stops before reaching verification tests
- Use informational language ("informational", "partial chain") rather than error language ("error", "warning")
- Respect pack-added chain patterns (e.g., HAZ -> REQ -> ARCH for ISO 26262)

## Acceptance Criteria

1. Given REQ-001 with links to ARCH-001 -> DDD-001 -> STORY-001 -> UT-001, running `specflow trace REQ-001` displays the full chain as an indented tree
2. Given a REQ with no downstream links, `specflow trace REQ-002` shows the REQ alone with no downstream entries
3. Given a REQ with `complies_with: ISO26262-3.7`, `specflow trace REQ-003` shows the standard clause at the top of the upstream chain
4. Running `specflow artifact-lint --type chain-report` produces a distribution summary without exit code 1 (informational only)
5. Given a HAZ artifact (pack-added type) linked to REQ -> ARCH -> STORY -> QT, the chain report includes it in the depth distribution
