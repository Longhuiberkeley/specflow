# Pack-Author Extraction Quality

The `/specflow-pack-author` skill **shall** use a deterministic extraction protocol when ingesting standards documents, minimizing hallucination risk and ensuring clause fidelity.

## Extraction Protocol

The skill **shall** instruct the AI agent to:

1. Extract the table of contents first, presenting it as a selection menu to the user
2. Scope extraction to user-selected sections only
3. Chunk by section boundaries (not page ranges), preserving hierarchical clause numbering
4. Constrain LLM output to `{id, title, description}` formatting only per chunk
5. Run a deduplication pass after extraction to merge overlapping fragments from adjacent sections

## Verification Layer

After extraction completes, the skill **shall**:

1. Spot-check 2-3 random source sections against extracted clauses to confirm no clauses were missed
2. Flag sections where extracted clause count does not match visible headings
3. Emit a `# VERIFY:` comment in the generated YAML documenting which sections were spot-checked

## Platform Awareness

The skill **shall** include guidance that:
- If the AI platform has native PDF reading capability, use it directly
- If the platform cannot read PDFs, ask the user to paste text or provide a URL

## Acceptance Criteria

1. Given a 100-page PDF standard, the skill instructs the agent to extract TOC first, then scope extraction to selected sections only
2. Given extracted clauses, the skill runs a verification step that spot-checks at least 2 source sections and reports any discrepancies
3. Given adjacent sections with overlapping clause fragments, the skill runs a dedup pass that merges duplicates
4. Given a platform without PDF support, the skill falls back to URL or pasted-text ingestion without error
5. The generated standards YAML includes `# VERIFY:` comments indicating which sections were spot-checked
