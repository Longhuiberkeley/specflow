# Standards Gap Analysis v2

The system **shall** provide enhanced standards gap analysis with actionable remediation guidance.

## Current State

`specflow standards gaps` lists uncovered clauses from installed standards. It does not score coverage, suggest remediation, or provide a summary dashboard.

## Requirements

The enhanced gap analysis **shall**:

1. **Coverage scoring**: compute a compliance score (0-100%) for each installed standard based on the ratio of covered to total clauses
2. **Remediation suggestions**: for each uncovered clause, suggest which artifact type (REQ, ARCH, DDD) should be created to address it
3. **Priority ranking**: sort gaps by clause severity or criticality (if defined in the standard pack)
4. **Summary dashboard**: display a per-standard summary showing total clauses, covered, uncovered, and score
5. **Machine-readable output**: support `--json` flag for integration with external dashboards

## Acceptance Criteria

1. `specflow standards gaps` shows per-standard coverage score as a percentage
2. Uncovered clauses are sorted by severity/priority
3. Each uncovered clause includes a suggested artifact type for remediation
4. `--json` flag produces machine-readable output
5. All operations are deterministic (zero AI tokens)
6. All existing tests continue to pass

> **Note**: Target version v0.3.0. Deferred from v0.2.0 to keep scope manageable.
