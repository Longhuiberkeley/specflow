# ReqIF Round-Trip Quality

The system **shall** improve ReqIF import/export quality for reliable round-trip interchange with DOORS, Polarion, and other ALM tools.

## Current Gaps

1. The `reqif_metadata` field is used during import to preserve unmapped ReqIF attributes, but is not listed in the requirement schema `optional_fields`, causing schema validation warnings
2. Export generates new UUIDs on each run instead of stable, deterministic UUIDs based on artifact content
3. No automated tests exist for the ReqIF import/export path
4. Export only covers requirements; architecture, design, and test artifacts are excluded

## Requirements

The system **shall**:

1. Add `reqif_metadata` to the requirement schema `optional_fields` list
2. Generate deterministic UUIDs for exported ReqIF elements based on artifact ID and content hash (same artifact always produces same UUID)
3. Provide a pytest test suite for ReqIF import and export covering:
   - Round-trip preservation of core fields (id, title, status, body)
   - Round-trip preservation of `reqif_metadata` (unmapped attributes survive import then export)
   - Deterministic UUID generation (export twice, UUIDs match)
   - Import of a valid ReqIF archive produces correct REQ artifacts
4. Extend export to support architecture (`ARCH-*`) and detailed-design (`DDD-*`) artifact types

## Acceptance Criteria

1. `reqif_metadata` appears in `.specflow/schema/requirement.yaml` optional_fields
2. Exporting the same artifact twice produces identical UUIDs in the ReqIF output
3. At least 4 tests in `tests/test_reqif.py` pass: round-trip fields, metadata preservation, deterministic UUID, import correctness
4. `specflow export --adapter reqif` exports ARCH and DDD artifacts alongside REQ
5. All existing tests continue to pass
