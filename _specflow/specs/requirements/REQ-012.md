# Compliance Score Dashboard

The system **shall** extend the `specflow status` command to provide coverage metrics and compliance scoring.

## Current State

`specflow status` shows artifact counts by type and status. It does not compute coverage ratios or compliance health indicators.

## Requirements

The `specflow status` output **shall** include a coverage section showing:

1. **REQ coverage**: percentage of approved REQs that have at least one STORY implementing them
2. **STORY test coverage**: percentage of implemented stories that have UT, IT, and QT artifacts linked via `verified_by`
3. **Gate pass rate**: percentage of phase-gate checklists that have passed (if any exist)
4. **Chain completeness**: percentage of approved specs with full V-model chains reaching a verification test

The dashboard **shall**:

- Use only existing artifact data (no new data structures required)
- Compute all metrics deterministically (zero-token)
- Display as a summary table below the existing artifact counts
- Exit code 0 always (informational, not blocking)

## Acceptance Criteria

1. `specflow status` displays a coverage summary section after artifact counts
2. REQ coverage shows percentage of approved REQs with linked STORY
3. STORY test coverage shows percentage of implemented stories with UT/IT/QT links
4. Chain completeness shows percentage of approved specs with verification tests
5. All metrics compute deterministically with zero AI tokens
6. Exit code is always 0 (informational)
7. All existing tests continue to pass
