# Compliance Evidence Reports

The system **shall** generate compliance evidence reports from the `/specflow-ship` workflow.

## Context

Regulated industries (medical, automotive, aerospace) require documented evidence that requirements were implemented, tested, and verified. This evidence is typically compiled into a Design History File (DHF), Technical File, or similar compliance dossier.

## Requirements

The `specflow ship` command **shall** support generating a compliance evidence report containing:

1. **Traceability matrix**: REQ → ARCH → DDD → UT/IT/QT chain for all verified requirements
2. **Test results summary**: pass/fail status of all linked test artifacts
3. **Baseline snapshot**: the baseline created during shipping, with diff from previous baseline
4. **Standards compliance**: for each installed standard, which clauses are covered and which have gaps
5. **Audit trail**: timestamps and author information for all status transitions

The report **shall**:

- Be output as a Markdown file in `.specflow/baselines/` alongside the baseline
- Use only deterministic data from existing artifacts (zero-token)
- Be suitable for inclusion in a DHF/Technical File submission package
- Be optional — triggered by a `--evidence` flag on `specflow ship`

## Acceptance Criteria

1. `specflow ship --evidence` generates a Markdown evidence report
2. Report includes traceability matrix for all verified REQs
3. Report includes test results summary
4. Report includes standards compliance coverage
5. Report is placed alongside the baseline in `.specflow/baselines/`
6. All operations are deterministic (zero AI tokens)
7. All existing tests continue to pass

> **Note**: Target version v0.3.0. Deferred from v0.2.0 to keep scope manageable.
