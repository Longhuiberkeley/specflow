# V-Model Test Scaffolding and Execute Enhancement

The system **shall** provide deterministic test artifact scaffolding from implemented specification artifacts, and enhance the `/specflow-execute` workflow to use it.

## CLI Command: specflow generate-tests

A new `specflow generate-tests` command **shall** create UT, IT, and QT stub artifacts from implemented spec artifacts:

1. **Type mapping** (deterministic):
   - DDD artifacts produce UT stubs (unit tests)
   - ARCH artifacts produce IT stubs (integration tests)
   - REQ artifacts produce QT stubs (qualification tests)

2. **Stub contents** (all deterministic, zero-token):
   - Correct `--type` (unit-test, integration-test, qualification-test)
   - Derived title: "Test \<spec title\>"
   - `verified_by` link to the source spec artifact
   - Placeholder body with section headers matching the spec type
   - For QT stubs: acceptance criteria from the REQ body copied as test case placeholders

3. **Scope modes**:
   - `specflow generate-tests --from DDD-001` — generate stubs for a single spec
   - `specflow generate-tests` (no args) — generate stubs for all specs missing verification pairs
   - `--dry-run` — show what would be created without writing files

4. **Duplicate prevention**: skip specs that already have a linked test of the correct type

## Execute Enhancement

The system **shall** update the `/specflow-execute` SKILL.md Step 5 to:

1. Call `specflow generate-tests` as a deterministic pre-step to create stubs with valid link structure
2. Then guide the AI to populate test cases from acceptance criteria
3. Run `specflow artifact-lint --type coverage` to verify all V-model pairs exist

## Acceptance Criteria

1. `specflow generate-tests --from DDD-001` creates a UT stub with correct type, title, links, and placeholder body
2. `specflow generate-tests --from REQ-001` creates a QT stub with acceptance criteria as test case placeholders
3. `specflow generate-tests` with no args creates stubs for all specs missing verification pairs
4. `--dry-run` shows what would be created without writing files
5. Existing tests are not duplicated on repeated runs
6. `/specflow-execute` SKILL.md Step 5 references `specflow generate-tests` as the stub creation mechanism
7. All operations are deterministic (zero AI tokens)
8. All existing tests continue to pass
