# Compliance Preset Packs

The system **shall** provide pre-built industry compliance pack schemas that users can install to extend SpecFlow's artifact types and checklists for regulated industries.

## Context

Regulated industries (automotive, medical, aerospace, finance) require compliance with specific standards. While `/specflow-pack-author` allows users to create packs from their own standard documents, many teams lack the expertise to design good pack schemas. Preset packs provide vetted, ready-to-use schemas for common standards.

## Requirements

### 1. Preset Pack Catalog

The system **shall** ship with preset packs for:

| Standard | Industry | Artifact Types Added |
|----------|----------|---------------------|
| ISO 26262 | Automotive | HAZ (hazard), SG (safety goal), SR (safety requirement) |
| IEC 62304 | Medical | RISK (risk analysis), SW-SAFETY (software safety) |
| DO-178C | Aerospace | SCA (software criticality analysis), TEST-PROC (test procedure) |
| SOX | Finance | CONTROL (internal control), AUDIT-FINDING |

### 2. Pack Installation

`specflow init --preset <name>` **shall**:
1. Copy preset schema files into `.specflow/schema/`
2. Create preset artifact type directories in `_specflow/specs/`
3. Copy preset checklist templates into `.specflow/checklists/`
4. Register new artifact types in `.specflow/config.yaml`

### 3. Pack Structure

Each preset pack **shall** contain:

```
presets/iso26262/
├── pack.yaml              # Pack metadata, artifact type definitions
├── schemas/               # Schema files for added artifact types
│   ├── hazard.yaml
│   ├── safety_goal.yaml
│   └── safety_requirement.yaml
├── checklists/            # Compliance checklists
│   └── hazard-analysis.yaml
└── templates/             # Default artifact templates
    └── hazard-template.md
```

### 4. Gap Analysis Integration

Preset packs **shall** include a default clause mapping:
- Each preset defines which standard clauses are relevant
- Gap analysis uses preset clause definitions when the pack is active
- Users can override or extend clause mappings

### 5. Documentation

Each preset pack **shall** include:
- README explaining the standard and artifact types
- Example artifacts showing proper usage
- Checklist explaining when each artifact type is needed

## Acceptance Criteria

1. `specflow init --preset iso26262` creates HAZ, SG, SR directories and schemas
2. `specflow create --type hazard` works after installing ISO 26262 preset
3. `specflow standards-gaps` uses preset clause definitions for gap analysis
4. Each preset includes at least 3 example artifacts
5. Preset installation is idempotent (re-running does not duplicate schemas)
6. Users can install multiple presets simultaneously (e.g., ISO 26262 + IEC 62304)
7. All existing tests continue to pass after preset installation
