# Test Suite Foundation

The system **shall** provide a `pytest` test suite covering the core library modules with deterministic, zero-token tests.

## Scope

The initial test suite **shall** cover:

1. `lib/artifacts.py` -- Artifact parsing, ID validation, fingerprint computation, link traversal (find_orphans, find_missing_v_pairs, trace_chain)
2. `lib/lint.py` -- Schema validation, status hierarchy, acceptance criteria detection
3. `lib/rbac.py` -- Role resolution, transition authorization, independence checking, CODEOWNERS rendering
4. `lib/standards.py` -- Standards loading, compliance gap analysis
5. `lib/baselines.py` -- Baseline creation, immutability, diff computation

## Test Infrastructure

The test suite **shall**:

- Use `pytest` as the test runner
- Use `tmp_path` fixtures for filesystem-based tests (no global state)
- Create sample artifact files in temp directories to test parsing and validation
- Run deterministically with no network or LLM calls
- Be runnable via `uv run pytest`

## Acceptance Criteria

1. Running `uv run pytest` discovers and executes all tests in `tests/`
2. Tests for `lib/artifacts.py` cover: parsing YAML frontmatter, computing fingerprints, finding orphans, finding missing V-model pairs
3. Tests for `lib/rbac.py` cover: solo-dev direct path, role-based authorization, independence violation detection
4. Tests for `lib/standards.py` cover: loading standards YAML, reporting covered vs uncovered clauses
5. All tests pass deterministically without environment variables, network, or LLM calls
6. `pytest` is listed as a development dependency in `pyproject.toml`
