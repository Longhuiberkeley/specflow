# P7: Team & Enterprise

> **Stub** — Detailed design to be fleshed out when P5-P6 are near completion.

## Goal

Make SpecFlow viable for multi-developer teams and enterprise compliance requirements: role-based access, test execution records, CI integration, and standards interchange.

## Rough Scope

### RBAC via Git

- Role configuration in `.specflow/config.yaml` (verifiers, safety_mgr, approvers)
- CODEOWNERS mapping for `_specflow/` directories
- Pre-commit hook reads git author + config roles, blocks unauthorized status transitions
- GPG-signed commits for status -> verified (electronic signature)
- PR templates auto-generated by `specflow init`
- Independence enforcement: implementer != verifier at PR level

### Draft IDs + CI renumbering

- Feature branches use readable draft IDs (e.g., `REQ-AUTH-a7b9`)
- On merge to main, `specflow-sequence` assigns sequential integers
- `yq` rewrites all links across repo in single merge commit
- Eliminates ID collision conflicts between branches

### Defect lifecycle

- DEF-* artifacts: open -> investigating -> fixing -> verified -> closed
- Links: `fails_to_meet` (broken requirement), `exposed_by` (catching test)
- On closure: reactive challenge engine extracts prevention pattern
- Dashboard integration in `specflow-status`

### Test execution records

- Default: CI owns execution, repo owns summaries in baselines
- Opt-in: `_specflow/test-runs/TR-*.md` for regulated audit trail
- CI pipeline step updates test summary via `specflow-baseline update-test-summary`

### CI integration

- Additional job alongside standard build/test/lint
- Two-pass: programmatic checks (zero tokens) then LLM-judged (token cost)
- Informational status badge by default, optionally blocking via branch protection

### ReqIF import/export

- `specflow import reqif` / `specflow export reqif` shell scripts
- Mechanical transformation between SpecFlow YAML and ReqIF XML
- Enables interchange with DOORS, Polarion

## Dependencies

- P5 (team execution orchestration)
- P6 (compliance hooks for RBAC enforcement)
